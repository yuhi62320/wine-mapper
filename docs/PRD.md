## 1. プロダクト概要

### 1.1 ビジョン
「ワインを飲みながら世界を旅する」— 一杯のワインからその産地の風景、畑、シャトーへと旅ができる体験を提供する。飲んだワインで世界地図が解放され、生産者の場所・風景・ストーリーに触れることで、まるで現地を訪れたかのようなワクワク感を生み出す。

### 1.2 ターゲットユーザー
- ワイン初心者〜中級者で、飲んだワインを記録したい人
- コレクション・探索が好きな人
- 香りや味の表現を学びながら記録したい人

### 1.3 コアバリュー
1. **簡単記録**: ラベル撮影→自動認識→ワンタップ登録
2. **豊かな表現**: アロマホイールベースの香り・味わい記録
3. **ワインで旅する**: 生産者の場所・畑・シャトーの写真を見て、産地を「訪れる」体験
4. **地図コレクション**: Fog of War方式で産地を探索
5. **成長実感**: ゲーミフィケーションによるモチベーション維持

## 2. 機能要件

### 2.1 ワインログ機能（コア）

#### 2.1.1 ラベルスキャン
- カメラでワインラベルを撮影
- OCR + AI によるテキスト抽出
- ワインDBと照合して情報を自動入力
- ユーザーが確認・編集して保存

#### 2.1.2 ワイン基本情報
| フィールド | 必須/任意 | 入力方法 |
|---|---|---|
| ワイン名 | 必須 | 自動入力 / 手入力 |
| ヴィンテージ（年） | 任意 | 自動入力 / 選択 |
| タイプ（赤/白/ロゼ/スパークリング/フォーティファイド/デザート/オレンジ） | 必須 | 選択 |
| ブドウ品種 | 任意 | 自動入力 / タグ選択 |
| 生産者名 | 任意 | 自動入力 / 手入力 |
| 生産者ホームページURL | 任意 | 自動入力 / 手入力 |
| 生産者所在地（座標） | 任意 | 自動取得（後述）/ 手動ピン設定 |
| 産地（国/地域/サブリージョン/アペラシオン） | 必須（国のみ） | 自動入力 / 階層選択 |
| アルコール度数 | 任意 | 自動入力 / 数値入力 |
| 価格 | 任意 | 数値入力 |
| ラベル写真 | 任意 | カメラ撮影 |

#### 2.1.3 生産者情報（ドメーヌ / シャトー / ワイナリー）

ワインをログする際、生産者名から以下の情報を自動取得・紐付けする。
これにより「飲みながらその場所を旅する」体験を実現する。

**生産者プロフィール**
| フィールド | 説明 | データソース |
|---|---|---|
| 生産者名 | ドメーヌ / シャトー / ワイナリー名 | ラベルスキャン / 手入力 |
| ホームページURL | 生産者の公式サイト | DB照合 / 手入力 |
| 所在地座標（緯度・経度） | 畑 / シャトー / ワイナリーの位置 | Google Places API / Wikidata / 手動ピン |
| 住所 | 所在地の住所テキスト | Google Places API |
| 国 / 地域 | 産地階層との紐付け | 座標からの逆ジオコーディング |

**生産者ロケーションビュー（ワインで旅する体験のコア）**
- マップ上に生産者の正確な位置をピン表示
- ピンをタップすると生産者詳細カードが開く:
  - **Google Maps埋め込み / リンク**: 実際の場所をMapで確認できる
  - **写真ギャラリー**: 畑、シャトー、ワイナリーの外観・風景写真
    - Google Places API のフォト機能で自動取得
    - ユーザーが自分の写真を追加することも可能
  - **ストリートビュー**: 対応地域であればGoogle Street Viewで現地を仮想散歩
  - **ホームページリンク**: 生産者公式サイトへのリンクボタン
  - **そこで飲んだワイン一覧**: 同じ生産者のログ済みワインを表示

**座標の取得方法（優先順位）**
1. **Google Places API**: 生産者名 + 地域で検索 → 座標・住所・写真を一括取得
2. **Wikidata SPARQL**: ワイナリーのエンティティから座標取得
3. **ユーザー手動設定**: マップ上でピンを置いて座標を指定
4. **コミュニティ蓄積**: 他ユーザーが登録した座標を共有DB経由で取得

#### 2.1.4 テイスティング記録

**香り（アロマ）— こだわりポイント**
- UC Davis アロマホイールベースの3階層分類
  - Tier 1: 大カテゴリ（果実、花、スパイス、植物、土、木、ナッツ等 10カテゴリ）
  - Tier 2: サブカテゴリ（柑橘系、トロピカル、赤い果実 等 約35種）
  - Tier 3: 具体的記述子（レモン、ライム、グレープフルーツ 等 200+種）
- アロマの由来分類（プライマリー / セカンダリー / ターシャリー）
- 日本語・英語バイリンガル表示
- 日本独自の表現にも対応（柚子、山椒、紫蘇、出汁、海苔 等）
- カスタム記述子の追加機能
- **UX**: タグ選択（メイン） + インタラクティブホイール（発見用） + フリーテキスト

**味わい（パレット）**
| 項目 | スケール |
|---|---|
| 甘さ | Bone Dry → Dry → Off-Dry → Medium-Sweet → Sweet → Lusciously Sweet |
| 酸味 | Low → Medium- → Medium → Medium+ → High |
| タンニン（赤のみ） | Low → Medium- → Medium → Medium+ → High |
| ボディ | Light → Medium- → Medium → Medium+ → Full |
| 余韻の長さ | Short → Medium- → Medium → Medium+ → Long |
| 全体評価 | 5段階（★） |

**テイスティングノート**
- フリーテキストでの自由記述

#### 2.1.5 飲んだ日時・シチュエーション
- 日付（デフォルト: 今日）
- 一緒に飲んだ人（任意タグ）
- 場所（レストラン名等、任意）

### 2.2 ワインマップ機能（差別化ポイント）

#### 2.2.1 Fog of War マップ
- 初期状態: ワイン産地が霧がかった灰色で表示
- ワインをログすると、その産地の霧が晴れてカラー表示に
- 色の濃さ = その産地で飲んだワインの本数

#### 2.2.2 ズームレベル
| レベル | 表示内容 | インタラクション |
|---|---|---|
| 世界 | 国単位で色分け + 完了率リング | タップで国にズーム |
| 国 | 地域/アペラシオン単位で色分け | タップで地域詳細パネル |
| 地域 | サブリージョン + 個別ワインピン | タップでワインカード |
| 生産者 | ドメーヌ/シャトー/ワイナリーのピン | タップで生産者ロケーションビュー |

#### 2.2.3 生産者ロケーションビュー（マップ最深部）
マップの最深ズームレベルで、実際の生産者の場所を体験できる。

- **生産者ピン**: 座標が登録された生産者をマップ上にピンで表示
- **ピンタップ → 生産者カード**:
  - 生産者名 + ロゴ / 写真
  - 畑 / シャトー / ワイナリーの写真ギャラリー（Google Places API自動取得 + ユーザー投稿）
  - Google Street View（対応地域のみ）で仮想散歩
  - 公式ホームページへのリンクボタン
  - その生産者のワインで自分がログしたもの一覧
  - 「Google Mapで開く」ボタン（外部アプリ連携）

#### 2.2.4 マップ統計
- 国ごと: 探索済み地域数 / 全地域数
- 地域ごと: ログ本数、品種数、平均評価、初回・最新ログ日
- 「次に探索すべき地域」のサジェスト

#### 2.3.3 バッジ
**産地探索系**
- Tour de France — フランス5/10/15/20地域制覇
- Italian Journey — イタリア5/10/15/20地域制覇
- New World Explorer — 新世界5カ国のワインをログ
- Globe Trotter — 10/20/30/40カ国のワインをログ

**ブドウ品種系**
- Noble Six — 6大品種制覇（CS, Merlot, PN, Chard, SB, Riesling）
- Grape Explorer — 10/25/50/100品種ログ
- Indigenous Vines — 固有品種5種ログ

**スタイル系**
- Bubble Enthusiast — スパークリング5本
- Rosé-Colored Glasses — ロゼ10本
- Sweet Tooth — デザートワイン5本
- Fortified Fan — フォーティファイド4種（Port, Sherry, Madeira, Marsala）

**マイルストーン系**
- First Sip — 初ログ
- Case Complete — 12本ログ
- Centurion — 100本ログ
- Vintage Collector — 5つの異なる年代のワインをログ

**ストリーク系**
- Weekly Regular — 4週連続ログ
- Dedicated Taster — 12週連続ログ
- Year of Wine — 12ヶ月連続ログ（毎月1本以上）

#### 2.3.4 週間ストリーク
- 単位: 週に1本以上ログ（日次ではなく週次）
- ストリークフリーズ: 月1回無料で使用可能
- リカバリー猶予: 翌週水曜日まで遡ってログ可能

#### 2.3.5 チャレンジ
- 月替わり「注目リージョン」→ 2倍XP
- 季節イベント（ボジョレーヌーヴォー週間、夏のロゼチャレンジ等）

## 3. 非機能要件

### 3.1 パフォーマンス
- ラベルスキャン結果表示: 5秒以内
- 基本ログ操作: 10秒以内で完了可能
- マップ表示・操作: スムーズ（60fps目標）

### 3.2 オフライン対応
- ワインログはオフラインでも作成可能（後で同期）
- マップの基本表示はキャッシュで対応

### 3.3 データ
- ユーザーデータのエクスポート機能（CSV/JSON）
- バックアップ・復元機能

## 4. データソース戦略

### 4.1 ラベル認識パイプライン
1. Google Cloud Vision API → OCR テキスト抽出
2. API4AI Wine Recognition → ワインブランド・品種識別
3. Wine-Searcher API / Global Wine Score API → 詳細情報照合
4. Open Food Facts → バーコード補完

### 4.2 地理データ
- UC Davis AVA Boundary Dataset（米国、GeoJSON）
- Wikidata SPARQL（世界のワイン産地 + 座標）
- 独自の地域境界データ（主要産地から段階的に拡充）

### 4.3 生産者データ
- **Google Places API**: 生産者名 + 地域で検索 → 座標、住所、写真、営業情報を取得
  - Place Search → Place Details → Place Photos の3ステップ
  - 料金: 従量課金（$200/月の無料クレジットあり）
- **Google Street View Static API**: 座標からStreet View画像を取得
- **Wikidata**: 著名ワイナリーのエンティティから座標・公式URL取得
- **ユーザー投稿 + コミュニティDB**: 未登録生産者はユーザーが座標・URLを登録し、共有DBに蓄積

### 4.4 アロマ・テイスティングデータ
- UC Davis Wine Aroma Wheel ベースの独自分類体系
- WSET（Wine & Spirit Education Trust）のSATフォーマット参考
- 日本語対応記述子を独自追加
